# スコア計算ページ改善タスク (2026-06-12)

## 目的
`スコア/index.html`（季節イベ最大スコア計算）を「使いやすく・かっこよく・わかりやすく」改善する。
計算ロジック（score = (20,000 + 280,000 × 与ダメ/HP) × (1+ボーナス%) の逆算）は変更しない。

## 改善方針
### わかりやすく
- [x] ページにタイトル・説明が一切ないので、ヘッダーにタイトル＋一行説明を追加
- [x] 折りたたみ式の「使い方・計算式」ヘルプを追加
- [x] セルの3段表示（ボーナス%/必要HP%/必要ダメージ）の凡例を追加
- [x] 必要HP% > 100% のセルを「不可」として赤系で明示（現状は普通に表示され誤読する）
- [x] 必要ダメージ ≤ 0（ノーダメで達成）のセルを「達成」として金色で明示（現状は負数が出る）
- [x] 入力値の下に 億/万 単位の読みを表示（100000000 → 1億）

### 使いやすく
- [x] ボーナス%入力時、ヘッダー直下に「必要ダメージ／必要HP%」の結果サマリーを大きく表示
- [x] セルをタップでそのボーナス%を選択＋必要ダメージをクリップボードへコピー（トースト通知）
- [x] 相手HP・目標Pt を localStorage に保存
- [x] resize 時は列数が変わるときだけ再描画
- [x] スマホで入力3つが窮屈なので、レスポンシブに段組み調整（HP/Pt 2列＋検索全幅）

### かっこよく
- [x] Tailwind CDN を廃止し、純CSS 1ファイル完結に（コンソール警告も消える）
- [x] ネオングリーン系テーマは維持しつつ、タイトルのグロー・検索ヒットのパルス発光・
      ホバー演出・背景の微細グリッドで質感を向上

## チェックリスト
